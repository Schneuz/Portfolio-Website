# Ziel
Jede Sektion soll in sich klar sein und gleichzeitig zum Gesamtsystem passen.

# Globale Regeln
- jede Sektion hat einen klaren Einstieg
- verwende konsistente Heading-Muster
- halte Textblöcke kurz
- nutze Cards, Linien und Negative Space systematisch
- vermeide unruhige Mischlayouts
- Desktop und Mobile getrennt mitdenken

# Hero
- 60/40 Split auf Desktop
- Text dominiert
- Bild ergänzt
- CTA sichtbar ohne Scroll-Zwang
- H1 maximal 2 Zeilen
- Meta unterhalb der CTA-Reihe

# Kurzprofil
- 2x2 Karten auf Desktop
- 1 Spalte auf Mobile
- vier Karten:
  - Wer bin ich
  - Was suche ich
  - Kernskills
  - Interessen

# Mindmap
- Desktop: echte Mindmap-Anmutung
- zentrale Node
- vier Hauptäste
- ruhige Verbindungslinien
- keine Interaktivität
- Mobile: Cluster-/Stack-Fallback

# Projekte
- konsistente Karten
- gleiche Informationsreihenfolge pro Karte
- Stack als Badges
- CTA klar benannt
- 2-spaltig oder 2x2 auf Desktop
- 1-spaltig auf Mobile

# Timeline
- Desktop: horizontal
- Mobile: vertikal/gestapelt
- eine ruhige Linie
- Dots zurückhaltend
- pro Eintrag nur Zeitraum, Titel, 1 kurze Einordnung

# Warum ich
- 3 Karten
- keine Floskeln
- kurze Titel + kurze Erklärungen

# Kontakt
- klarer Abschluss
- direkte Kontaktwege
- keine Formularpflicht in v1

# Qualitätsfragen
1. Ist die Sektion in 3 Sekunden verstehbar?
2. Ist die wichtigste Information visuell dominant?
3. Passt die Sektion stilistisch zum Rest?
4. Funktioniert sie auf Mobile genauso glaubwürdig?
